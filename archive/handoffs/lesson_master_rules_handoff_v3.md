## 2. プロジェクト概要

### 作るもの
教師向けHTML自動生成Webツール。教師がファイル（文型データ）をアップロードすると、以下の4つの出力物が自動生成される。

| 出力物 | 用途 | 受け手 |
|---|---|---|
| スライドHTML | 授業中に画面に映す | 学習者 |
| 宿題HTML | 授業後に学習者が使う | 学習者 |
| アクティビティHTML | 活動中に使うタスクシート | 学習者 |
| 教案docx | 授業準備に使う | 教師 |

### データの3層構造
```
lesson_NN.json（課マスター）
  └── 「この課で何を教えるか」文型・語彙・例文・フロー

session_NNN.json（セッション）
  └── 「この授業でどの文型をどの順で教えるか」

activity_catalog.json（活動カタログ）
  └── 全課共通の活動定義（現在18種類）
```

### 使用教科書と方針
**「日本語の教え方ABC」を唯一の基準教科書として採用（確定）。**
ABCの「教え方の例」を忠実に実装することがこのシステムの最大の差別化要因。
将来「みんなの日本語」「げんき」に完全移行する場合はこのシステムを使う意味がなくなる。
他教科書への移行が必要になった場合は `lesson_NN.json` の中身とT/Lスクリプトを差し替えるだけで対応可能（エンジン側は変えない）。

### 品質安定のための方針
**課マスターの質が安定しないと、その下にできるセッションファイルの質も安定しない。**
そのため課マスターの構造設計を上流から固めることを最優先としている。

## 3. 「日本語の教え方ABC」完全フォーマット仕様

**このセクションは第1〜30課のPDF精読から帰納した公式仕様。課マスター設計の基準。**

### 全体構造（全30課で例外なし）

```
各課 = セクション1（文型）
     + セクション2（文法知識の整理）
     + セクション3（教え方の例）
     + セクション4（活動例）
     + 補足要素（column・オンライン対応・プラスα・注意）
```

---

### セクション1：文型

- 番号付きリスト。1行程度の例文のみ。説明なし。
- 重要箇所に下線
- 1文型に複数バリエーションがある場合は枝番（2-1、2-2 等）
- 文型数：初級前半3〜6、初級後半4〜6、中上級3〜5

---

### セクション2：文法知識の整理

**対象：教師向けの文法解説（学習者向けではない）**

内容の種類：
- ●（黒丸）箇条書きで文法の本質・制約・誤用注意
- 課によって図表あり（活用表・分類表・視点マーク図等）
- セクション内にもプラスα・注意が登場することがある

課別の付随図表：

| 図表 | 主な登場課 |
|---|---|
| ます形活用表 | 第3課 |
| こそあど全体系表 | 第2課 |
| 場所助詞の用法整理表 | 第4・5課 |
| 形容詞活用表 | 第6課 |
| 授受動詞の関係表 | 第10課 |
| 視点マーク図 | 第5・10・25・28・29課 |
| 条件文の接続表 | 第22・27課 |
| 受け身・使役の変化表 | 第28・29課 |
| 敬語の対応表 | 第30課 |

---

### セクション3：教え方の例

#### サブセクション「〇〇の導入」のタイトル形式（必須）
```
[番号]　「〜文型〜」の導入
```

#### 本文の構成順序（固定）
```
1. 教材準備の記述（教具・既習事項の確認）
2. T/Lスクリプト
3. 板書図
4. 補足説明文
5. [プラスα] ボックス（任意）
6. [注意] ボックス（任意）
7. [PCスライドヒント] ボックス（任意）
8. [オンライン授業ヒント] ボックス（任意）
```

#### T/Lスクリプトの記法

| 記号 | 意味 |
|---|---|
| `T：` | 教師（Teacher） |
| `L：` | 学習者全体 |
| `A：` `B：` `C：` | 特定の学習者個人 |
| `〈〉` | 非言語行動・教具操作・ジェスチャー |
| **太字** | 教師が強調して言う部分・モデル文 |
| 下線 | 板書される文・ターゲット文型 |
| `……` | 学習者の発話を待つ間 |
| 🖊（鉛筆） | 板書を示す記号（Lの発話行末） |
| `〈リピート〉` | 全員で繰り返す指示 |

#### スクリプトの標準展開パターン（全課共通）
```
① 教師がアクション（実物・絵カードを見せる）
② 教師がモデル文を提示（太字）
③ 学習者全体がリピート（板書される→下線付き）
④ 教師が自分のことで言う（「わたしは〜」）
⑤ 個別の学習者に聞く（「Aさんは？」）
⑥ 学習者Aが答える
⑦ 他の学習者にリピートを促す
⑧ 別の語・別の人物に替えて繰り返す
⑨ 学習者同士のQ&Aへ発展
```

#### 板書図の仕様
- ホワイトボード枠＋文型骨格（ひらがな）＋🖊
- 助詞は丸で囲む：`〜(に)〜ます。`
- 語彙は入れない。骨格のみ。
- 変化矢印が入る場合あり（受け身・使役等）

#### 教具の種類（vocabType）

| vocabType | 説明 | 代表課 |
|---|---|---|
| `vocabulary_card` | 1語1枚の絵カード | 第1・3・4課 |
| `real_object` | 教師が実際のものを持ってくる | 第2・10課 |
| `scene_picture` | 複数要素が入った場面絵・地図 | 第7課 |
| `contrast_picture` | 2つを並べた対比絵 | 第6課 |
| `world_map` | 世界地図（国籍・国名導入用） | 第1課 |
| `calendar` | カレンダー・時制導入用 | 第3・5課 |
| `table_chart` | 表・グラフ（気温表等） | 第9課 |
| `family_chart` | 家系図（家族の呼称導入用） | 第10課 |

#### 画像の役割（imageRole）

| imageRole | 説明 |
|---|---|
| `vocab_person` | 人物を表す語彙カード（職業・属性） |
| `vocab_object` | ものを表す語彙カード |
| `scene` | 複数要素が入った場面絵 |
| `contrast` | 2つを対比させた絵 |
| `diagram` | 視点マーク・矢印図・文法図解 |

### セクション4：活動例

#### タイトル形式
```
● 活動例　「活動名」　[⬇]
```
⬇はDL素材がある課のみ

#### 本文の構成順序（固定）
```
1. 活動の全体説明（設定・目的・タスク概要）← 1〜3段落
2. 準備物の説明 ← 本文中に埋め込み（独立ブロックではない）
3. 手順の説明 ← 本文中に埋め込み
4. 【会話例】 ← 独立ブロック・A/B/Tの対話形式
5. [プラスα] ボックス（任意）
6. [注意] ボックス（任意）
7. [オンライン授業ヒント] ボックス（任意）
```

#### 活動例の必須フィールド（B-9ルール）
| フィールド | 内容 |
|---|---|
| 活動名 | 例：「会話『はじめまして』」 |
| 設定 | どんな場面か（1〜2文） |
| 準備物 | 何を何枚用意するか |
| 会話例 | 最低1往復 |
| fadingStage | 5段階のどこか（stage1〜5） |
| ABCとの対応 | activity_catalog.jsonのどのIDか |

---

### 補足要素の仕様

| 要素 | 目的 | 形式 |
|---|---|---|
| プラスα | 余裕があれば教える発展内容 | 角丸ラベル＋●箇条書き |
| 注意（cautionNote） | 誤用・落とし穴・制限事項 | 角丸ラベル＋●または段落 |
| オンライン授業ヒント | オンラインでの代替手段 | 点線ボックス＋ヘッドフォンアイコン |
| PCスライドヒント | PPT・アニメーション活用法 | 点線ボックス＋PCアイコン |
| column | 授業運営・教育論の補足 | 独立枠（一部課のみ） |

---

### 3原則（全30課精読から帰納）

1. **語彙先行の原則**：すべての課で、文型スクリプトに入る前に必ず語彙導入段落がある
2. **板書図は文型ごとに1枚**：各導入サブセクションに必ず1枚。骨格のみ、助詞を丸囲み
3. **活動例は課に1つ**：複数文型を統合した自由産出タスク。設定・準備物・手順・会話例のセットが必須

## 4. 確定済み：課マスター作成ルール全14項目

### A：必須ルール（これがないと課マスターが壊れる）

#### A-1　例文はPDF照合してから書く
- 教科書PDFをその課の分だけ読み通してから例文を書く
- 「先に草案を書いて後で照合」は禁止
- 照合済みであることを `originalSources` に記録する

#### A-2　登場人物を勝手に作らない
- 教科書に登場する固有名詞はそのまま使う
- それ以外は `Aさん/Bさん` 等の教科書記号を使う
- 独自キャラクター（田中先生・山田さん等）は作らない

#### A-3　教科書の固有名詞は変えない
- 変えた場合は必ず `originalSources.replacementNote` に理由を記録する

#### A-4　canDoは「〜できる」の形で1文型につき1文
- 形式：「〜できる」で統一
- 数：1文型につき1文のみ
- 良い例：「『これ・それ・あれ』で近くのものと遠くのものを指し示せる」
- 悪い例：「指示詞を使う。これ・それ・あれの違いがわかる」（2文・形式バラバラ）

#### A-5　語彙導入の教具を必ず記録する
語彙リストの各語に `vocabType` を付与する（セクション3の教具表を参照）。

#### A-6　画像の役割（imageRole）を例文・語彙ごとに明記する
`imageRole` フィールドを必須とする（セクション3のimageRole表を参照）。

---

### B：品質ルール（これがないと課によってバラつく）

#### B-5　語彙の記録のしかた（3点セット）
① `jlptLevel`：N5〜N1を書く
② `isFirstAppearance`：この課が初出か・前の課で習ったかを書く（`true`/`false`）
③ 絵カード化できない語（「わたし」「です」等）は理由を `_note` に書く

#### B-6　practiceTemplatesは最低2つ
- 1文型につき必ず2つ書く
- 標準構成：「肯定文テンプレート」＋「疑問文または否定文テンプレート」
- 形式：`＿＿＿` 形式の穴埋めで統一

#### B-7　materialNeeds[]の付与ルール
- `intro_activity` と `main_activity` には**必須**
- `pattern`・`example` はオプション
- `wrapUp`・`review`・`intro_slide` には付与しない

#### B-8　推奨時間の目安

| ステップの種類 | 目安の範囲 |
|---|---|
| 復習 | 3〜5分 |
| 導入アクティビティ | 5〜10分 |
| 文型1つ | 3〜7分 |
| メインアクティビティ | 10〜20分 |
| まとめ | 2〜5分 |

#### B-9　活動例の必須フィールド
ABCの活動例を取り込むとき、以下を必ず書く：
活動名・設定（1〜2文）・準備物・会話例（最低1往復）・fadingStage・ABCとの対応（catalog ID）

---

### C：運用ルール（作業効率に関わる）

#### C-9　作業ステップの順序
この順番を必ず守る：
1. **Step 1a**：骨格（文型の一覧と構造のみ）
2. **Step 1b**：語彙リスト（PDFを読んでから）
3. **Step 1c**：例文（PDFと照合しながら）
4. **Step 1d**：materialNeeds（例文が確定してから）
5. **Step 1e**：アクティビティ選定（ABCの活動例とcatalogを照合）

#### C-10　変更履歴の書き方
`_meta.changes` には「変更内容」と「変更理由」の両方を必ず書く。

#### C-11　バージョン管理の分離
- `formatVersion`：全課共通のスキーマ版数
- `lessonVersion`：この課だけの改訂番号（`1.0` から始める）
- 両方を `_meta` に並べて書く

#### C-12　教案の手順はABC準拠で書く
- 課マスターには `teachingHint`（1〜2文の要点）だけ書く
- T/Lスクリプトの詳細は教案docxに出力する
- 教案には「日本語の教え方ABC 第N課 教え方の例に準拠」と明記する

## 6. 確定済み：課マスターの設計方針

### フィールド命名の統一（確定）
| 旧フィールド名 | 新フィールド名 | 理由 |
|---|---|---|
| `note` | `cautionNote` | ABCの「注意」ラベルと対応させる |
| `duration` | `_recommendedDuration` | session優先の意味を明示（実装済み） |
| `flow[].minutes` | `flow[]._recommendedMinutes` | 同上（実装済み） |

### 廃止・削除するフィールド（確定）
| フィールド | 理由 |
|---|---|
| `activity` セクション全体 | 旧設計の遺産。現在は `flow[].activityId` で管理済み |

### 語彙情報の責務分離（確定）
- 語彙の詳細情報（vocabType・imageRole・jlptLevel等）は `vocabulary.byPattern[].words[]` のみで管理
- `flow[].materialNeeds[]` には「教具の種類と数」だけを書き、詳細は `vocabulary` を参照する設計

### バージョン管理の分離（確定）
- `_meta.formatVersion`：全課共通のスキーマ版数（現在 `2.7` が lesson_01 の最終値）
- `_meta.lessonVersion`：この課だけの改訂番号（`1.0` から始める）

## 7. 確定済み：フィールド仕様（完全版）

### vocabulary.byPattern[].words[] の新規フィールド

```json
{
  "word": "先生",
  "reading": "せんせい",
  "en": "teacher",
  "jlptLevel": "N5",
  "isFirstAppearance": true,
  "vocabType": "vocabulary_card",
  "imageRole": "vocab_person",
  "_note": ""
}
```

| フィールド | 型 | 必須 | 内容 |
|---|---|---|---|
| `word` | string | ✅ | 表記 |
| `reading` | string | ✅ | ひらがな読み |
| `en` | string | ✅ | 英訳 |

## 10. PDF精読全データ：第1〜10課の完全記録

### 第1課　名詞

**文型一覧**

| ID | 文型 | 板書形式 | JLPT |
|---|---|---|---|
| p1 | 〜は〜です（人物） | 〜は　〜です。 | N5 |
| p2 | 〜ですか（はい/いいえ）/ だれ | 〜ですか。はい、〜です。／いいえ、〜じゃありません。 | N5 |
| p3 | 〜の〜（所属） | 〜の　〜です。 | N5 |

**語彙リスト**

p1・p2共通（7語）：先生、医者、会社員、学生、男の人、女の人、外国人
p3専用（5語）：会社、学校、銀行、大学、デパート

**教科書原典例文**

| no | 文型 | 例文 |
|---|---|---|
| 1 | p1 | 鈴木さんは先生です。 |
| 2-1 | p2 | リンさんは先生ですか。 |
| 2-2 | p2 | はい、先生です。／いいえ、先生じゃありません。 |
| 2-3 | p2 | だれですか。―キムさんです。 |
| 3-1 | p3 | Aさんは会社の会社員です。 |
| 3-2 | p3 | Bさんは学校の先生です。 |
| 3-3 | p3 | Aさんはさくら銀行の会社員です。 |
| 3-4 | p3 | リンさんはさくら大学の学生です。 |
| 3-5 | p3 | Aさんはさくらデパートの会社員です。 |

**語彙導入の教具**
- p1・p2：語彙カード（人物・職業絵カード）＋世界地図
- p3：語彙カード（所属機関の建物絵カード）

**導入手順の核心**
- p1：人物絵カード→「鈴木さんです」→「鈴木さんは先生です」→世界地図で国籍→リピート→学習者の実名で置換練習
- p2：同じ絵カードで「〜ですか。はい/いいえ」Q&A形式。有名人写真で「だれですか」を導入
- p3：自分の胸を指して「わたし。これはわたしのかばんです」→所属機関カードと組み合わせて導入

**文法知識（教師向け）**
- A＝Bの関係になる「〜は〜です」に限定。「父は会社です」のような例は出さない
- 否定は「じゃありません」と「ではありません」の両方を教えてよい
- 過去形「でした」・普通体「だ」にはこの段階で触れない

**活動例：会話「はじめまして」**
- 設定：学習者2人ペアで自己紹介をする
- 準備物：簡単な名刺カード（やや大きめ）、会話モデルのスライドまたは絵カード
- 手順：教師が一人二役でモデルを示す→学習者2人ずつペアになって自己紹介を繰り返す。相手を次々と変える
- 会話例：「はじめまして。リンです。中国人です。どうぞよろしく。」「はじめまして。ケリーです。アメリカから来ました。どうぞよろしく。」「ケリーさんは先生ですか。」「はい、教師です。リンさんは？」「わたしは学生です。東西大学の学生です。」
- fadingStage：stage4〜5

**注意（ABCより）**
- 絵カードの人物については「だれですか」でよいが、クラスメートには「どなたですか」を使う
- 導入時は教師が質問のモデルを示すとき、すぐに答えを言わず学習者の反応（うなずき・首振り）を待ってから答える

**プラスα（ABCより）**
- 「〜から来ました」は自己紹介表現としてそのまま教える
- AもBも教師の場合は「わたしも教師です」という「も」の使い方を教える

## 11. 全課共通パターン（PDF精読から帰納）

### 「教え方の例」の構造（全10課で例外なし）

### 全課を通じた教育原則（PDF精読から帰納）

1. **語彙先行の原則**：文型より語彙を先に導入する
2. **数限定の原則**：語彙数・動詞数などを初めは少なくしぼる（第3課：動詞5つ、第7課：位置詞6語）
3. **実物優先の原則**：可能であれば絵カードより実物を使う（第2・10課等）
4. **対比の原則**：形容詞導入では対比実物または対比絵を必ず使う（第6課）
5. **視点マークの原則**：視点が関わる文型は視点マークで視覚化（第5・10課）
6. **モデル先行の原則**：教師が先に自分のことを言ってから学習者に同じことを聞く
7. **リピート→Q&A→ペアワークの原則**：すべての文型導入でこの順に展開
8. **会話例セットの原則**：活動例には必ず会話例がついている
9. **プライバシー配慮の原則**：個人的なことについて言わせるときは強制しない（第6・10課）
