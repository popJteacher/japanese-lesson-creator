## §2. v2.9の問題根本原因分析（v5.0新規）

### 2-1. 最重要：色品質の劣化（Double-Hex Anchoring の消失）

v2.9 では Imagen v4.6 の「hex値を画像内テキストとして描画するバグ」への対応として STYLE_RECIPE から hex 値を完全除去した。

```python
# v2.5 まで（良かった）― Double-Hex Anchoring 三重指定
"soft cream white background (similar to #FBFBFB, hex value FBFBFB)"

# v2.9（色品質劣化）― 色名のみ
"soft cream white background"
```

この変更が色精度・全体的な雰囲気の劣化をもたらした最大要因と判断。

### 2-2. アスペクト比問題

v2.9 の設計前提：「アスペクト比は GAS API パラメータで制御するためプロンプトに記述不要」

手動生成（Gemini直接入力）では GAS を経由しないため：
- テンプレートA〜J：`vocabulary card illustration` という文言を Gemini が 1:1 と解釈 → **正常**
- テンプレートK〜T：そのような文脈文言がなく、Gemini が **16:9 をデフォルト採用** → 崩壊

### 2-3. v2.6〜v2.9 で蓄積した問題の全体像

| 変更 | バージョン | 問題 |
|---|---|---|
| hex値除去 | v2.8/v2.9 | 色品質劣化（最大の原因） |
| テンプレートK〜T追加 | v2.6 | アスペクト比未指定・語彙カード文脈なし |
| CHARACTER_RENDERING_POLICY 複雑化 | v2.7 | 肌色指定が分岐設計になり一貫性が低下 |
| GRAMMAR_PATTERNS_C 追加 | v2.6 | 例文画像設計の方針が旧設計と並存したまま未決定 |
| STYLE_RECIPE のプロンプト分離 | v2.7 | A〜J と K〜T で組み立て手順が異なり複雑化 |
| プロンプト長の増大 | v2.7〜 | Gemini が重要要素を落とすリスク増加 |

### 2-4. lesson_01/02 に K〜T テンプレートは不要という確認

lesson_01/02 の vocabulary の imageRole は以下のみ：

```
lesson_01: vocab_person × 12語 → テンプレートA
           vocab_object × 5語  → テンプレートB（建物）

lesson_02: vocab_object × 16語 → テンプレートD（具体物）またはB（建物）
           vocab_person × 2語  → テンプレートA
```

**K〜T（counter/time/transportation/family/weather/sensory/adverb 等）は lesson_01/02 では一切不要。**
v2.6〜v2.9 で追加した拡張はすべて、この2課の S列生成には無関係だった。

## §4. 手動生成時の追加ルール（v5.0新規・GAS非使用時に適用）

v2.5.1 は GAS API 経由の生成を前提としている。Gemini に直接プロンプトを投入する手動生成の場合、以下を**プロンプト先頭に追加**すること：

```
Square 1:1 aspect ratio. ONE single character or object only, centered in the frame.
Do not generate multiple figures.
```

GAS 経由の場合はこの追加不要（アスペクト比は API パラメータで制御）。

### 7-2. lesson_01/02 S列プロンプト生成の手順

```
1. lesson_NN.json の vocabulary.byPattern から全語彙を読み込む
2. 各語彙の imageRole で使用テンプレートを決定:
   - vocab_person  → テンプレートA（vocabulary_person）
   - vocab_object  → 建物系は B（vocabulary_building）/ 具体物系は D（vocabulary_object_concrete）
3. v2.5.1 の該当テンプレートにプロンプトを埋め込む
4. [PURPOSE] 等のセクションヘッダーを除去してフラット文字列に変換
5. §0-A チェック（プレースホルダー残留・hex# なし・日本語混入なし）
6. JSON形式で出力:
   { "word_医者": "...", "word_先生": "...", ... }
```

### 7-3. 建物 vs 具体物の判定基準（lesson_02 用）
