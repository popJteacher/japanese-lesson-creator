## §13. 画像生成ライン確定事項【v4.2 新設・追記のみ・削除禁止】

### 確定した方針

**Google Imagen 4 課金 API を使用する。Kaggle/FLUX ラインは採用しない。**

確定日：2026-05-18

### 判断根拠

**Kaggle/FLUX を採用しない理由：**

| 問題 | 詳細 |
|---|---|
| 量産速度が致命的に遅い | 1枚5分以上。375件で31時間超。Kaggle GPU 枠（週30h）をほぼ全消費。N5全語彙（約800枚）なら3週間以上。 |
| 画風問題が未解決 | simplevectorflux v2 との相性・512→1024リサイズによる品質劣化・HEXカラーコードの効かなさ。デバッグコストが見えない。 |
| プロンプト設計の互換性なし | master_prompt_design_guide_v2_5.py は Gemini 向けに最適化されており、FLUX 向けに全面書き直しが必要。 |
| スタイル統一リスク | 画風問題が解決できなかった場合、生成済み枚数をすべて作り直す羽目になる。 |

**Imagen 4 課金 API を使う理由：**

| 理由 | 詳細 |
|---|---|
| GASコードの変更点が1箇所のみ | モデル名を `imagen-4.0-generate-001`（または `imagen-4.0-fast-generate-001`）に変更するだけ。既存パイプラインを流用できる。 |
| プロンプト設計がそのまま使える | master_prompt_design_guide_v2_5.py は Gemini/Imagen 向けに設計済み。即使用可能。 |

```
以下のテーマについて、認知科学・第二言語習得・グラフィックデザインの観点から調査してください。

【調査テーマ】
フラットベクターイラスト（テキストなし・静止画）で表現する
日本語中級文法パターン（授受・比較・条件・許可・義務・モダリティ）の視覚化戦略

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ1：授受表現（あげます・もらいます・くれます）L10
━━━━━━━━━━━━━━━━━━━━━━━━━━━
核心課題：あげます・もらいます・くれますの「方向性」が視覚化の本質。同じ「物のやりとり」でも動詞によって矢印の向き・恩恵の受け取り方が変わる。

調査事項：
1. 授受・方向性を持つ行為（贈与・受取・恩恵）の視覚化に使われる記号体系
2. 矢印の色・方向・スタイルで「あげる/もらう/くれる」を区別するデザイン手法
3. 言語教材（EFL/ELT・日本語教育）での授受表現の視覚化先行事例

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ2：比較表現（〜より・いちばん）L9
━━━━━━━━━━━━━━━━━━━━━━━━━━━
調査事項：
1. 比較（2者）・最上級（3者以上）の視覚化に使われるグラフィックパターン（棒グラフ・サイズ差・矢印・ランキング順序など）
2. フラットデザインの制約内（グラデーション不可・影不可）で差を表現する手法
3. 「どちらのほうが」という2択選択と「いちばん」という1位選出の構図の違い

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ3：モダリティ（許可・禁止・義務・不必要）L18
━━━━━━━━━━━━━━━━━━━━━━━━━━━
対象文型：〜てもいいですか・〜てはいけません・〜なければなりません・〜なくてもいいです

調査事項：
1. 許可・禁止・義務・不必要を表すユニバーサルデザイン・ピクトグラムの標準（ISO・JIS等）
2. 言語教材でモダリティを視覚化している先行事例（特に日本語教材）
3. ✓（許可）・✗（禁止）以外で「義務」「不必要」を表す視覚記号
4. 同一の動作に対して「OK/NG/必須/任意」を1枚の絵の中に示す構図

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ4：思考・発話引用（と思います・と言いました）L16
━━━━━━━━━━━━━━━━━━━━━━━━━━━
調査事項：
1. 思考バブル・発話バブル・伝聞を区別する国際的なグラフィック規範
2. フラットベクタースタイルでの吹き出し（speech bubble）デザインの先行事例
3. 「そうです（伝聞）」「かもしれません（可能性）」「でしょう（推測）」のモダリティの強度差を視覚的に区別する手法
4. 不確実性・推測の度合いを視覚化するデザインシステム

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ5：条件・仮定（〜たら・〜ば・〜なら）L22
━━━━━━━━━━━━━━━━━━━━━━━━━━━
調査事項：
1. 因果・条件関係を静止画で表現するグラフィック手法（矢印・コマ・分岐など）
2. 言語教材での条件文・仮定法の視覚化先行事例
3. 「〜たら」（実現後の条件）と「〜ば」（仮定条件）の違いを視覚的に示す手法
4. 分岐構造（もし〜なら→A、もし〜でなければ→B）のフローチャート的表現の教材適用事例

━━━━━━━━━━━━━━━━━━━━━━━━━━━
グループ6：経験・時制の複雑化（〜たことがあります・〜たほうがいいです）L15

1. 過去経験・回想・記憶を表す視覚記号（タイムライン・星印・記憶フラッシュ等）
2. 「やったことがある」というチェックマーク型の経験カウンター表現の言語教材での事例
3. 「〜したほうがいい」という助言の方向性の視覚化手法

━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━
調査事項：
1. 同時進行する2動作を静止画1枚で表現する構図手法（分割・重ね・矢印2本等）
2. 「進行中」と「結果の状態」を視覚的に区別するデザイン記号
3. 〜ています（職業・習慣）「田中さんは東京に住んでいます」を静止画で示す手法

【全グループ共通の調査事項】
```
