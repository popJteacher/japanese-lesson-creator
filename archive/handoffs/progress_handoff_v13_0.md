### 1-2. マスタープロンプトガイド v2.5 vs v2.9.1 の差分比較

アップロードされた v2.4 / v2.5 / v2.9.1 の三版を実ファイルで突き合わせ、
`建物テンプレート（Template B）および BUILDING_CUES が三版とも完全に同一`
であることを機械検証で確認。

**主な発見事項（次チャットへの前提知識）：**
- Template B の看板指示は三版とも `A single short **Japanese** signage word` — つまり**どの版でも英語を明示許可した版は存在しない**
- 旧画像（vocab_銀行 "BANK"、vocab_大学 "UNIVERSITY CAMPUS"）が英語だったのは、プロンプトに日本語看板を指定したところを画像生成器が英語に勝手に置換した結果（偶然の産物）
- v2.9.1 は v2.5.1 で「不採用」と決めた v2.6〜v2.9 拡張スタックそのものであり、ガイド自己矛盾（問題D）の状態だった

### 1-3. マスタープロンプトガイド v3.0 の作成

**ベース：v2.5（v2.5.1 が回帰先として確定した検証済みベースライン）**
**バージョン：`master_prompt_design_guide_v3_0.py`**

解決した問題（progress_handoff_v12_0 §1-2 の問題A〜F＋内部矛盾）：

| 問題 | 内容 | 対処 |
|---|---|---|
| 問題A | 国籍人物カードの旗巨大化 | 問題Eの根本解決（NATIONALITY_NOUN_POLICY新設）で恒久化 |
| 問題B | 建物カードの文字化け | 建物サイネージを英語短語1語に確定（BANK/UNIVERSITY等） |
| 問題C | 建物カードの情報過多・空パネル量産 | 3アンカー→単一primary_scene_cue・反クラッター文追加 |
| 問題D | ガイド本体の方針矛盾（v2.5回帰が宣言されていたのにv2.9.1が現行だった） | v2.5を正系ベースとして確定。v2.6〜v2.9.1の拡張スタックを全面破棄 |
| 問題E | 国籍名詞の視覚化戦略欠落 | **PART 1.1 NATIONALITY_NOUN_POLICY 新設**（v2.5にはなかった） |
| 問題F | 色指定の世代間矛盾（hex三重指定 vs 色名のみ） | hex三重指定を全37箇所除去・色名のみに統一 |
| 内部矛盾 | グローバル制約「文字絶対禁止」とTemplate B「日本語看板1語許可」の矛盾 | グローバル制約に建物サイネージ例外を明示的に切り出し整合化 |

**v2.6〜v2.9.1 の拡張の採否（v3.0ガイドに明記済み）：**
- hex完全除去 → **採用**（問題F解決）
- テキストラベル絶対禁止の文言強化 → **部分採用**（建物例外を整合させた上で）
- CHARACTER_RENDERING_POLICY / ASPECT_RATIO_POLICY / その他大型追加 → **全て不採用**（退行の原因だったため）

**v3.0で意図的に扱わなかった事項（改訂履歴§3に明記）：**
- generic_* の肌色「warm medium固定」問題 → 設計方針の変更に該当。人間判断が必要

## §5. 重要な設計決定事項（次チャット担当者への引き継ぎ）

### 5-1. ガイドバージョンの確定系統

```
正系: v2.5 → v3.0（このチャットで確定）
廃棄: v2.6 / v2.7 / v2.8 / v2.9 / v2.9.1（退行を確認・全て破棄）
今後の改訂は v3.0 を唯一のベースとして行う
```

### 5-2. 建物画像の確定仕様（v3.0）

- 看板：英語短語1語のみ（日本語看板は廃止。文字化けが構造的に確定するため）
- シーンキュー：primary_scene_cue 1行のみ（旧3アンカーは廃止）
- 「Negative space in the top one-third for typography」は絶対に書かない
- 帯テキスト（語ラベル）はGAS側オーバーレイの責務

### 5-3. 国籍名詞の確定仕様（v3.0）

- 人物が主役（70-80%）、国旗は小ピン≤4%
- 「PRIMARY / sole identifier / 10-12% / facing forward」は絶対に書かない
- 旗上に文字なし

### 5-4. 色指定の確定方針（v3.0）

- 「色名のみ」で統一。`similar to #XXXXXX` や `hex value XXXX` は書かない
- Imagen の テキスト描画バグ・トークン浪費の回避

### 5-5. 固定キャラ（char_*）の位置づけ

- 語彙カードプロンプトファイル（word_*）とは別ファイルで管理
- 現在は v8 由来の画像が存在するが v3.0 スタイルと乖離の可能性あり
