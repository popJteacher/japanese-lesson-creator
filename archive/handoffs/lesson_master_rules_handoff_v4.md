## 2. プロジェクト概要

### 作るもの
教師向けHTML自動生成Webツール。教師がファイル（文型データ）をアップロードすると、以下の4つの出力物が自動生成される。

| 出力物 | 用途 | 受け手 |
|---|---|---|
| スライドHTML | 授業中に画面に映す | 学習者 |
| 宿題HTML | 授業後に学習者が使う | 学習者 |
| アクティビティHTML | 活動中に使うタスクシート | 学習者 |
| 教案docx | 授業準備に使う | 教師 |

### データの3層構造
```
lesson_NN.json（課マスター）
  └── 「この課で何を教えるか」文型・語彙・例文・フロー

session_NNN.json（セッション）
  └── 「この授業でどの文型をどの順で教えるか」

activity_catalog.json（活動カタログ）
  └── 全課共通の活動定義（現在18種類）
```

### 使用教科書と方針
**「日本語の教え方ABC」を唯一の基準教科書として採用（確定）。**
ABCの「教え方の例」を忠実に実装することがこのシステムの最大の差別化要因。
他教科書への移行が必要になった場合は `lesson_NN.json` の中身とT/Lスクリプトを差し替えるだけで対応可能。

### 品質安定のための方針
**課マスターの質が安定しないと、その下にできるセッションファイルの質も安定しない。**
そのため課マスターの構造設計を上流から固めることを最優先としている。

## 3. 「日本語の教え方ABC」完全フォーマット仕様

**※ v3 セクション3と完全に同一。変更なし。**

### 全体構造（全30課で例外なし）

```
各課 = セクション1（文型）
     + セクション2（文法知識の整理）
     + セクション3（教え方の例）
     + セクション4（活動例）
     + 補足要素（column・オンライン対応・プラスα・注意）
```

### セクション1：文型
- 番号付きリスト。1行程度の例文のみ。説明なし。
- 重要箇所に下線
- 1文型に複数バリエーションがある場合は枝番（2-1、2-2 等）
- 文型数：初級前半3〜6、初級後半4〜6、中上級3〜5

### セクション2：文法知識の整理
**対象：教師向けの文法解説（学習者向けではない）**

課別の付随図表：

| 図表 | 主な登場課 |
|---|---|
| ます形活用表 | 第3課 |
| こそあど全体系表 | 第2課 |
| 場所助詞の用法整理表 | 第4・5課 |
| 形容詞活用表 | 第6課 |
| 授受動詞の関係表 | 第10課 |
| 視点マーク図 | 第5・10・25・28・29課 |
| 条件文の接続表 | 第22・27課 |
| 受け身・使役の変化表 | 第28・29課 |
| 敬語の対応表 | 第30課 |

### セクション3：教え方の例

#### T/Lスクリプトの記法

| 記号 | 意味 |
|---|---|
| `T：` | 教師（Teacher） |
| `L：` | 学習者全体 |
| `A：` `B：` `C：` | 特定の学習者個人 |
| `〈〉` | 非言語行動・教具操作・ジェスチャー |
| **太字** | 教師が強調して言う部分・モデル文 |
| `……` | 学習者の発話を待つ間 |

#### 教具の種類（vocabType）

| vocabType | 説明 | 代表課 |
|---|---|---|
| `vocabulary_card` | 1語1枚の絵カード | 第1・3・4課 |
| `real_object` | 教師が実際のものを持ってくる | 第2・10課 |
| `scene_picture` | 複数要素が入った場面絵・地図 | 第7課 |
| `contrast_picture` | 2つを並べた対比絵 | 第6課 |
| `world_map` | 世界地図（国籍・国名導入用） | 第1課 |
| `calendar` | カレンダー・時制導入用 | 第3・5課 |
| `table_chart` | 表・グラフ（気温表等） | 第9課 |
| `family_chart` | 家系図（家族の呼称導入用） | 第10課 |

#### 画像の役割（imageRole）

| imageRole | 説明 |
|---|---|
| `vocab_person` | 人物を表す語彙カード（職業・属性） |
| `vocab_object` | ものを表す語彙カード |
| `scene` | 複数要素が入った場面絵 |
| `contrast` | 2つを対比させた絵 |
| `diagram` | 視点マーク・矢印図・文法図解 |

### セクション4：活動例

#### 活動例の必須フィールド（B-9ルール）
| フィールド | 内容 |
|---|---|
| 活動名 | 例：「会話『はじめまして』」 |
| 設定 | どんな場面か（1〜2文） |
| 準備物 | 何を何枚用意するか |
| 会話例 | 最低1往復 |
| fadingStage | 5段階のどこか（stage1〜5） |
| ABCとの対応 | activity_catalog.jsonのどのIDか |

### 3原則（全30課精読から帰納）

1. **語彙先行の原則**：すべての課で、文型スクリプトに入る前に必ず語彙導入段落がある
2. **板書図は文型ごとに1枚**：各導入サブセクションに必ず1枚。骨格のみ、助詞を丸囲み
3. **活動例は課に1つ**：複数文型を統合した自由産出タスク。設定・準備物・手順・会話例のセットが必須

## 4. 確定済み：課マスター作成ルール全14項目

**※ v3 セクション4と完全に同一。変更なし。**

### A：必須ルール

#### A-1　例文はPDF照合してから書く
#### A-2　登場人物を勝手に作らない
#### A-3　教科書の固有名詞は変えない
#### A-4　canDoは「〜できる」の形で1文型につき1文
#### A-5　語彙導入の教具を必ず記録する
#### A-6　画像の役割（imageRole）を例文・語彙ごとに明記する

### B：品質ルール

#### B-5　語彙の記録のしかた（3点セット）
① `jlptLevel`：N5〜N1を書く
② `isFirstAppearance`：初出か既習かを書く（`true`/`false`）
③ 絵カード化できない語は理由を `_note` に書く

#### B-6　practiceTemplatesは最低2つ
#### B-7　materialNeeds[]の付与ルール
#### B-8　推奨時間の目安

| ステップの種類 | 目安の範囲 |
|---|---|
| 復習 | 3〜5分 |
| 導入アクティビティ | 5〜10分 |
| 文型1つ | 3〜7分 |
| メインアクティビティ | 10〜20分 |
| まとめ | 2〜5分 |

#### B-9　活動例の必須フィールド
活動名・設定（1〜2文）・準備物・会話例（最低1往復）・fadingStage・ABCとの対応（catalog ID）

### C：運用ルール

#### C-9　作業ステップの順序（必須）
1. Step 1a：骨格
2. Step 1b：語彙リスト（PDF読んでから）
3. Step 1c：例文（PDFと照合しながら）
4. Step 1d：materialNeeds（例文確定後）
5. Step 1e：アクティビティ選定（ABCの活動例とcatalogを照合）

#### C-10〜C-12　変更履歴・バージョン管理・教案記述
（v3と同一）

**⚠️ アクティビティHTMLと教案docxで「手順」の内容が異なる点に注意：**

| 対象 | 出力先 | 文体・視点 | 詳細度 |
|---|---|---|---|
| 学習者向け手順（`playerSteps[]`） | アクティビティHTML | 「〜ましょう」（学習者視点） | 最大3ステップ |

**第1課の記述例：**

```
【教案docx（教師向け）】
教師が一人二役でモデルを示す → 学習者2人ずつペアになって
自己紹介を繰り返す。相手を次々と変える。

【アクティビティHTML（学習者向け）playerSteps[]】
① 名前と仕事を伝えましょう
② 相手に質問しましょう
③ 次のパートナーに変わりましょう
```

### アクティビティHTMLのUI仕様（v4確定）

| 機能 | 仕様 | 根拠 |
|---|---|---|
| `furiganaToggle` | あり（宿題HTMLと統一） | 既存設計の踏襲 |
| `englishToggle` | **あり（v4新規確定）** | 口頭説明が伝わらない場合の安全網 |

**`englishToggle` 採用の判断根拠：**
オンライン1対1授業で教師の口頭説明（英語）が伝わらなかった場合、
学習者が `englishToggle` をONにすれば `playerSteps[].en` を自己確認できる。
HTMLの設計で言語の壁をカバーするという設計思想と整合。

```json
      }
    ]
  }

```

#### `playerSteps[]` フィールド仕様（v4新規確定）

| フィールド | 型 | 必須 | 内容 |
|---|---|---|---|
| `playerSteps` | array | ✅ | 学習者向け手順（最大3要素） |
| `playerSteps[].ja` | string | ✅ | 日本語の手順（「〜ましょう」形） |
| `playerSteps[].en` | string | ✅ | 英語訳（englishToggle用） |

#### `playerSteps[]` 記述ルール（v4確定）

| ルール | 内容 |
|---|---|
| 上限 | **最大3ステップ**（認知負荷管理） |
| 文末 | 「〜ましょう」または「〜します」で統一 |
| 視点 | **学習者の行動のみ**（教師の操作・教具の指示は書かない） |
| 文型への言及 | **禁止**（「〜は〜ですを使って」は書かない。文型は会話例で示す） |
| 英訳 | **必須**（`en` フィールドに自然な英語で） |
